## Overview

OpsChat's AI Services provide intelligent features powered by Groq's LLM API. These endpoints enable conversation summarization, real-time translation, and context-aware reply suggestions to enhance team communication.

<Note>
  All AI endpoints are currently **unauthenticated** and do not require the `x-user-id` header. However, rate limiting may apply.
</Note>

## Summarize Conversation

<Card title="POST /api/ai/summarize" icon="compress">
  Generate a concise 2-3 sentence summary of a chat conversation, focusing on key decisions and action items.
</Card>

### Model Details

- **Model**: `llama-3.3-70b-versatile`
- **Temperature**: 0.5 (balanced between creativity and consistency)
- **Use Case**: Meeting recaps, conversation digests

### Request

<ParamField body="messages" type="array" required>
  Array of message strings to summarize. Can include multiple messages from a conversation.
</ParamField>

### Response

<ResponseField name="summary" type="string">
  AI-generated summary of the conversation (2-3 sentences)
</ResponseField>

### Code Example

<CodeGroup>
```bash cURL
curl -X POST https://api.opschat.com/api/ai/summarize \
  -H "Content-Type: application/json" \
  -d '{
    "messages": [
      "Alice: We need to finalize the API design by Friday",
      "Bob: I can have the endpoints documented by Wednesday",
      "Alice: Perfect, that gives us two days for review",
      "Charlie: I will start implementing the auth middleware today",
      "Bob: Great! Let me know if you need the specs early"
    ]
  }'
```

```javascript JavaScript
const summarizeConversation = async (messages) => {
  const response = await fetch('https://api.opschat.com/api/ai/summarize', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ messages })
  });
  
  const { summary } = await response.json();
  return summary;
};

// Usage
const messages = [
  'Alice: We need to finalize the API design by Friday',
  'Bob: I can have the endpoints documented by Wednesday',
  'Alice: Perfect, that gives us two days for review',
  'Charlie: I will start implementing the auth middleware today'
];

const summary = await summarizeConversation(messages);
console.log(summary);
// Output: "The team agreed to finalize the API design by Friday. 
//          Bob will document endpoints by Wednesday, giving two days 
//          for review. Charlie will begin implementing auth middleware today."
```

```python Python
import requests

def summarize_conversation(messages):
    response = requests.post(
        'https://api.opschat.com/api/ai/summarize',
        json={'messages': messages}
    )
    return response.json()['summary']

messages = [
    'Alice: We need to finalize the API design by Friday',
    'Bob: I can have the endpoints documented by Wednesday',
    'Alice: Perfect, that gives us two days for review'
]

summary = summarize_conversation(messages)
print(f"Summary: {summary}")
```
</CodeGroup>

### Response Example

```json
{
  "summary": "The team agreed to finalize the API design by Friday. Bob will document endpoints by Wednesday, giving two days for review. Charlie will begin implementing auth middleware today."
}
```

<Expandable title="400 - Invalid Messages">

  ```json
  }
  ```
  
  Returned when `messages` is not provided or is not an array.
</Expandable>

  ```json

    "error": "AI service failed"
  }
  ```
  
  Returned when the Groq API is unavailable or returns an error.
</Expandable>

## Translate Text

<Card title="POST /api/ai/translate" icon="language">
  Translate text into any target language using AI. Returns only the translated text without explanations.
</Card>

### Model Details

- **Model**: `llama-3.1-8b-instant`
- **Temperature**: 0.3 (optimized for accuracy)
- **Use Case**: Multi-language team communication

### Request

<ParamField body="text" type="string" required>
  The text to translate
</ParamField>

<ParamField body="lang" type="string" required>
  Target language (e.g., "Spanish", "French", "Japanese", "German")
</ParamField>

### Response

<ResponseField name="translation" type="string">
  Translated text in the target language
</ResponseField>

### Code Example

<CodeGroup>
```bash cURL
curl -X POST https://api.opschat.com/api/ai/translate \
  -H "Content-Type: application/json" \
  -d '{
    "text": "Hello, how are you today?",
    "lang": "Spanish"
  }'
```

```javascript JavaScript
const translateText = async (text, targetLang) => {
  const response = await fetch('https://api.opschat.com/api/ai/translate', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ text, lang: targetLang })
  });
  
  const { translation } = await response.json();
  return translation;
};

// Usage examples
const spanish = await translateText('Hello, how are you today?', 'Spanish');
console.log(spanish); // "Hola, ¿cómo estás hoy?"

const french = await translateText('The meeting starts at 3 PM', 'French');
console.log(french); // "La réunion commence à 15h"

const japanese = await translateText('Thank you for your help!', 'Japanese');
console.log(japanese); // "ご協力ありがとうございます！"
```

```python Python
import requests

def translate_text(text, target_lang):
    response = requests.post(
        'https://api.opschat.com/api/ai/translate',
        json={'text': text, 'lang': target_lang}
    )
    return response.json()['translation']

# Examples
spanish = translate_text('Hello, how are you today?', 'Spanish')
print(spanish)  # "Hola, ¿cómo estás hoy?"

german = translate_text('The project is complete', 'German')
print(german)  # "Das Projekt ist abgeschlossen"
```
</CodeGroup>

### Response Example

```json
  "translation": "Hola, ¿cómo estás hoy?"
}
```

## Suggest Smart Replies

<Card title="POST /api/ai/suggest-replies" icon="comment-dots">
  Generate 3 contextual reply suggestions based on recent conversation history.
</Card>

### Model Details

- **Model**: `llama-3.1-8b-instant`
- **Temperature**: 0.6 (balanced for natural responses)
- **Context Window**: Last 5 messages
- **Use Case**: Quick responses, mobile keyboards

### Request

<ParamField body="context" type="array" required>
  Array of recent message strings. The last message is considered the most recent. Only the last 5 messages are used for context.
</ParamField>

### Response

<ResponseField name="suggestions" type="array">
  Array of 3 suggested reply strings (5-10 words each, casual but professional)
</ResponseField>

### Code Example

<CodeGroup>
```bash cURL
curl -X POST https://api.opschat.com/api/ai/suggest-replies \
  -H "Content-Type: application/json" \
  -d '{
    "context": [
      "Alice: Can you review the PR when you get a chance?",
      "Bob: Sure! I will take a look this afternoon",
      "Alice: Thanks! There are some tricky changes in the auth module",
      "Bob: No problem, I will be thorough"
    ]
  }'
```

```javascript JavaScript
const getSuggestedReplies = async (conversationContext) => {
  const response = await fetch('https://api.opschat.com/api/ai/suggest-replies', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ context: conversationContext })
  });
  
  const { suggestions } = await response.json();
  return suggestions;
};

// Usage
const context = [
  'Alice: Can you review the PR when you get a chance?',
  'Bob: Sure! I will take a look this afternoon',
  'Alice: Thanks! There are some tricky changes in the auth module',
  'Bob: No problem, I will be thorough'
];

const replies = await getSuggestedReplies(context);
console.log(replies);
// ["I appreciate it!", "Let me know if you have questions", "Sounds good, thanks!"]

// Display as quick reply buttons
replies.forEach((reply, index) => {
  const button = document.createElement('button');
  button.textContent = reply;
  button.onclick = () => sendMessage(reply);
  quickRepliesContainer.appendChild(button);
});
```

```python Python
import requests

def get_suggested_replies(context):
    response = requests.post(
        'https://api.opschat.com/api/ai/suggest-replies',
        json={'context': context}
    )
    return response.json()['suggestions']

context = [
    'Alice: Can you review the PR when you get a chance?',
    'Bob: Sure! I will take a look this afternoon',
    'Alice: Thanks! There are some tricky changes in the auth module'
]

suggestions = get_suggested_replies(context)
for i, suggestion in enumerate(suggestions, 1):
    print(f"{i}. {suggestion}")
```
</CodeGroup>

### Response Example

```json
{
  "suggestions": [
    "I appreciate it!",
    "Let me know if you have questions",
    "Sounds good, thanks!"
  ]
}
```

### Fallback Behavior

If the AI service fails or returns invalid JSON, the endpoint provides default suggestions:

```json
{
  "suggestions": ["👍", "Sounds good", "Can you clarify?"]
}
```

### Error Responses

<Expandable title="400 - Invalid Context">
  ```json
  {
    "error": "Invalid context"
  }
  ```
  
  Returned when `context` is not provided, not an array, or is empty.
</Expandable>

<Info>
  This endpoint gracefully handles failures by returning fallback suggestions instead of error responses, ensuring a smooth user experience.
</Info>

## Rate Limiting

<Warning>
  AI endpoints may be subject to rate limiting based on the Groq API quotas. Consider implementing:
  
  - Client-side caching for translations
  - Debouncing for smart reply requests
  - Exponential backoff on 429 responses
</Warning>

### Recommended Limits

- **Summarization**: 10 requests per minute per user
- **Translation**: 30 requests per minute per user
- **Smart Replies**: 20 requests per minute per user

## Best Practices

<CardGroup cols={2}>
  <Card title="Context Quality" icon="messages">
    For best results with summarization and smart replies, provide clear, well-formatted messages with speaker names.
  </Card>
  
  <Card title="Language Specificity" icon="language">
    Use specific language names ("Spanish" not "ES", "Chinese Simplified" vs "Chinese Traditional").
  </Card>
  
  <Card title="Error Handling" icon="circle-exclamation">
    Always implement fallbacks for AI endpoints, as external LLM services may have downtime.
  </Card>
  
  <Card title="Privacy Consideration" icon="shield">
    Be transparent with users that AI features process message content. Consider opt-in settings for sensitive channels.
  </Card>
</CardGroup>
