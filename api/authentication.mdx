## Overview

OpsChat uses a simple custom header-based authentication system. After registering and logging in, you'll receive user credentials that must be included in subsequent API requests.

<Warning>
Currently, OpsChat uses a simplified authentication system with the `X-User-Id` header. In production environments, consider implementing JWT tokens for enhanced security.
</Warning>

## Authentication Flow

1. **Register** a new account using the signup endpoint
2. **Login** to receive your user credentials
3. **Include** the `X-User-Id` header in all authenticated requests

---

## Endpoints

### Register New Account

Create a new user account with email and password.

<CodeGroup>
```bash cURL
curl -X POST https://api.opschat.com/api/signup \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "password": "securePassword123"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.opschat.com/api/signup', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    email: 'user@example.com',
    password: 'securePassword123'
  })
});

const data = await response.json();
console.log(data);
```

```python Python
import requests

response = requests.post(
    'https://api.opschat.com/api/signup',
    json={
        'email': 'user@example.com',
        'password': 'securePassword123'
    }
)

data = response.json()
print(data)
```
</CodeGroup>

#### Request Body

<ParamField body="email" type="string" required>
  User's email address. Must be unique in the system.
</ParamField>

<ParamField body="password" type="string" required>
  User's password. Will be securely hashed using bcrypt before storage.
</ParamField>

#### Response

<ResponseField name="success" type="boolean">
  Indicates whether the registration was successful
</ResponseField>

<ResponseField name="userId" type="integer">
  The unique identifier of the newly created user
</ResponseField>

<ResponseField name="email" type="string">
  The email address of the registered user
</ResponseField>

<CodeGroup>
```json 201 Success
{
  "success": true,
  "userId": 42,
  "email": "user@example.com"
}
```

```json 400 Bad Request
{
  "error": "Required fields missing"
}
```

```json 409 Conflict
{
  "error": "Email already exists"
}
```

```json 500 Server Error
{
  "error": "Internal server error"
}
```
</CodeGroup>

<Note>
New users are automatically added to the default workspace (ID: 1) with the MEMBER role upon successful registration.
</Note>

### Login

Authenticate with existing credentials to access your account.

<CodeGroup>
```bash cURL
curl -X POST https://api.opschat.com/api/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "password": "securePassword123"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.opschat.com/api/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    email: 'user@example.com',
    password: 'securePassword123'
  })
});

const data = await response.json();
// Store user.id for subsequent authenticated requests
localStorage.setItem('userId', data.user.id);
```

```python Python
import requests

response = requests.post(
    'https://api.opschat.com/api/login',
    json={
        'email': 'user@example.com',
        'password': 'securePassword123'
    }
)

data = response.json()
# Store user_id for subsequent requests
user_id = data['user']['id']
```
</CodeGroup>

#### Request Body

<ParamField body="email" type="string" required>
  User's registered email address
</ParamField>

<ParamField body="password" type="string" required>
  User's password
</ParamField>

#### Response

<ResponseField name="success" type="boolean">
  Indicates whether the login was successful
</ResponseField>

<ResponseField name="user" type="object">
  User information object
  
  <ResponseField name="user.id" type="integer">
    Unique user identifier (use this for the `X-User-Id` header)
  </ResponseField>
  
  <ResponseField name="user.email" type="string">
    User's email address
  </ResponseField>
  
  <ResponseField name="user.username" type="string">
    User's username (may be null if profile not set up)
  </ResponseField>
  
  <ResponseField name="user.name" type="string">
    User's display name (may be null if profile not set up)
  </ResponseField>
</ResponseField>

<ResponseField name="isProfileSetup" type="boolean">
  Indicates whether the user has completed their profile setup (username is set)
</ResponseField>

<CodeGroup>
```json 200 Success
{
  "success": true,

    "email": "user@example.com",

    "name": "John Doe"

}
```

```json 200 Success (Profile Not Setup)

  "success": true,
  "user": {
    "id": 42,
    "email": "user@example.com",
    "username": null,
    "name": null
  },
  "isProfileSetup": false
```

```json 400 Bad Request
{
  "error": "Required fields missing"
}
```

```json 401 Unauthorized

```

```json 500 Server Error
  "error": "Internal server error"
```

## Making Authenticated Requests

After logging in, include the `X-User-Id` header in all API requests that require authentication.

### Authentication Header

<ParamField header="X-User-Id" type="string" required>
  The user ID obtained from the login response. Must be included in all authenticated requests.
</ParamField>

### Example Authenticated Request

<CodeGroup>
```bash cURL
curl -X GET https://api.opschat.com/api/channels \
  -H "X-User-Id: 42"
```

```javascript JavaScript
const response = await fetch('https://api.opschat.com/api/channels', {
  headers: {
    'X-User-Id': '42'
  }
});

const channels = await response.json();
```

```python Python
import requests

response = requests.get(
    'https://api.opschat.com/api/channels',
    headers={
        'X-User-Id': '42'
    }
)

channels = response.json()
```
</CodeGroup>

### Authentication Middleware

The authentication middleware validates the `X-User-Id` header on protected routes:

```javascript src/middleware/authMiddleware.js:3
const authMiddleware = (req, res, next) => {
    const userId = req.headers['x-user-id'];

    if (!userId) {
        return res.status(401).json({ error: 'Authentication required' });
    }

    req.user = {
        userId: parseInt(userId)
    };

    next();
};
```

### Check Username Availability

Before updating a profile, check if a username is available.

<CodeGroup>
```bash cURL

  -H "Content-Type: application/json" \
  -d '{
    "username": "johndoe"
  }'
```

## Error Responses

All authentication endpoints follow consistent error response patterns:

### Status Codes

| Status Code | Description |
|-------------|-------------|
| `200` | Request successful |
| `201` | Resource created successfully |
| `400` | Bad request - missing required fields |
| `401` | Unauthorized - invalid credentials or missing authentication |
| `409` | Conflict - resource already exists (duplicate email/username) |
| `500` | Internal server error |

### Error Response Format

All error responses follow this structure:

```json
{
  "error": "Error message describing what went wrong"
}
```

<Warning>
When authentication fails with a 401 status code, you must re-authenticate by logging in again to obtain valid credentials.
</Warning>

## Security Best Practices

<CardGroup cols={2}>
  <Card title="Store Credentials Securely" icon="lock">
    Never expose user IDs in client-side code or URLs. Use secure storage mechanisms like localStorage or secure cookies.
  </Card>
  
  <Card title="Use HTTPS" icon="shield-check">
    Always use HTTPS in production to encrypt authentication data in transit.
  </Card>
  
  <Card title="Handle Errors Gracefully" icon="triangle-exclamation">
    Implement proper error handling for failed authentication attempts and redirect users to login when needed.
  </Card>
  
  <Card title="Validate Input" icon="circle-check">
    Always validate and sanitize user input on the client side before sending to the API.
  </Card>
</CardGroup>
