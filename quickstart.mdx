## Welcome!

This guide will get you from zero to your first chat message in under 5 minutes. We'll use Docker Compose for the simplest setup, then show you how to deploy to Kubernetes if you need production-grade infrastructure.

<Note>
  **Prerequisites**: Docker and Docker Compose installed on your machine
</Note>

## Option 1: Docker Compose (Recommended for First-Time Users)

Perfect for local development and testing. This will spin up all services with one command.

<Steps>
  <Step title="Clone the Repository">
    ```bash
    git clone https://github.com/yourusername/opschat.git
    cd opschat
    ```
  </Step>

  <Step title="Set Up Environment Variables">
    Create a `.env` file in the root directory:

    ```bash .env
    # Required: Get a free API key from https://console.groq.com
    GROQ_API_KEY=your_groq_api_key_here

    # Optional: Defaults work out of the box
    VITE_API_URL=http://localhost:3000
    ```

    <Tip>
      **Don't have a Groq API key?** Sign up for free at [console.groq.com](https://console.groq.com). The free tier is generous enough for testing.
    </Tip>
  </Step>

  <Step title="Start All Services">
    Run the following command to start OpsChat:

    ```bash
    docker-compose up
    ```

    This will start:
    - PostgreSQL database (port 5432)
    - Redis cache (port 6379)
    - MinIO object storage (port 9000)
    - Backend API (port 3000)
    - Frontend UI (port 8080)

    <Info>
      First run will take 2-3 minutes to build images. Subsequent starts are much faster.
    </Info>
  </Step>

  <Step title="Access OpsChat">
    Open your browser and navigate to:

    ```
    http://localhost:8080
    ```

    You should see the OpsChat login/signup page!
  </Step>

  <Step title="Create Your First Account">
    1. Click **Sign Up**
    2. Choose a username and password
    3. You're in! Try joining the `#general` channel
  </Step>

  <Step title="Send Your First Message">
    - Type a message in the `#general` channel
    - Try uploading an image
    - Click the translate icon to translate a message
    - Use the AI summarization button to summarize the conversation
  </Step>
</Steps>

<Check>
  **Success!** You now have OpsChat running locally with full AI features.
</Check>

## Option 2: Kubernetes with KIND (Advanced)

For users who want to test Kubernetes deployments locally or prepare for production.

<Warning>
  **Prerequisites**: Docker, kubectl, and KIND must be installed
</Warning>

<Steps>
  <Step title="Ensure Prerequisites">
    Install KIND (Kubernetes in Docker):

    <CodeGroup>
    ```bash macOS
    brew install kind kubectl
    ```

    ```bash Linux
    curl -Lo ./kind https://kind.sigs.k8s.io/dl/v0.20.0/kind-linux-amd64
    chmod +x ./kind
    sudo mv ./kind /usr/local/bin/kind
    ```

    ```bash Windows
    choco install kind kubectl
    ```
    </CodeGroup>
  </Step>

  <Step title="Clone and Configure">
    ```bash
    git clone https://github.com/yourusername/opschat.git
    cd opschat
    ```

    Create a `.env` file:
    ```bash .env
    GROQ_API_KEY=your_groq_api_key_here
    ```
  </Step>

  <Step title="One-Command Deployment">
    Use the automated deployment script:

    ```bash
    chmod +x deploy.sh
    ./deploy.sh
    ```

    This script will:
    - Create a KIND cluster
    - Install NGINX ingress controller
    - Create Kubernetes secrets
    - Build Docker images
    - Load images into KIND
    - Deploy all services

    <Info>
      The deployment takes about 3-5 minutes on the first run.
    </Info>
  </Step>

  <Step title="Verify Deployment">
    Check that all pods are running:

    ```bash
    kubectl get pods
    ```

    You should see:
    ```
    NAME                        READY   STATUS    RESTARTS   AGE
    backend-xxxxxxxxxx-xxxxx    1/1     Running   0          2m
    backend-xxxxxxxxxx-xxxxx    1/1     Running   0          2m
    frontend-xxxxxxxxxx-xxxxx   1/1     Running   0          2m
    postgres-xxxxxxxxxx-xxxxx   1/1     Running   0          2m
    redis-xxxxxxxxxx-xxxxx      1/1     Running   0          2m
    minio-xxxxxxxxxx-xxxxx      1/1     Running   0          2m
    ```
  </Step>

  <Step title="Access the Application">
    Open your browser to:

    ```
    http://localhost
    ```

    <Note>
      Kubernetes uses port 80 (no port number needed in the URL)
    </Note>
  </Step>
</Steps>

### Manual Kubernetes Deployment

If you prefer step-by-step control:

<Accordion title="Show Manual Steps">
  ```bash
  # 1. Create KIND cluster
  kind create cluster --config kind-config.yaml --name opschat-local

  # 2. Install ingress controller
  kubectl apply -f k8s/ingress-nginx.yaml
  kubectl wait --namespace ingress-nginx \
    --for=condition=ready pod \
    --selector=app.kubernetes.io/component=controller \
    --timeout=120s

  # 3. Create secrets
  kubectl create secret generic opschat-secrets \
    --from-literal=GROQ_API_KEY=your_key_here

  # 4. Build images
  docker build -t opschat-backend:v1 ./opschat-backend
  docker build -t opschat-frontend:v1 \
    --build-arg VITE_API_URL=http://localhost \
    ./opschat-frontend

  # 5. Load into KIND
  kind load docker-image opschat-backend:v1 --name opschat-local
  kind load docker-image opschat-frontend:v1 --name opschat-local

  # 6. Deploy all services
  kubectl apply -f k8s/
  ```
</Accordion>

    ```bash
    curl http://localhost:3000/health
    ```

    ```json

      "status": "healthy",

      "storage": "connected"
    ```

  </Accordion>

  <Accordion title="Check MinIO Storage" icon="box">
    Open MinIO console:
    ```
    http://localhost:9001
    ```

    Login: `admin` / `password123`

</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Port Already in Use" icon="triangle-exclamation">
    If you see `port is already allocated`, another service is using the port.

    **Docker Compose:**
    ```bash
    # Find what's using port 8080
    lsof -i :8080

    # Stop the conflicting service or change ports in docker-compose.yml
    ```
  </Accordion>

  <Accordion title="Database Connection Failed" icon="database">
    ```bash
    # Check if PostgreSQL is running
    docker ps | grep postgres

    # Check logs
    docker logs opschat_db

    # Restart services
    docker-compose restart
    ```
  </Accordion>

  <Accordion title="AI Features Not Working" icon="robot">
    Verify your `GROQ_API_KEY` is set correctly:

    ```bash
    # Check if env var is loaded
    docker exec opschat_api env | grep GROQ

    # Should show: GROQ_API_KEY=gsk_...
    ```

    If not set, update `.env` and restart:
    ```bash
    docker-compose down
    docker-compose up
    ```
  </Accordion>

  <Accordion title="File Upload Fails" icon="file-arrow-up">
    MinIO might not be initialized correctly:

    ```bash
    # Check MinIO logs
    docker logs opschat_storage

    # Recreate MinIO bucket
    docker-compose restart minio-init
    ```
  </Accordion>
</AccordionGroup>
